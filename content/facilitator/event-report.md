# The event report — for facilitators

The event report is the whole event in a single, self-contained HTML file —
every visible tool's results, formatted the way they look on screen: sorted
vote tallies, scatter plots, cluster groupings, PPCO cards, sketches and all.
It opens in any browser, needs no internet connection and no Samvinas
account, so you can email it to a client or drop it in a project folder as
the permanent record of the session.

## Creating it

1. Open your event's **Settings** (the gear on the session view).
2. Choose **Export Report**. Generation takes a moment while the data is
   gathered.
3. The file downloads as `<Event-Name>-report.html`.

For a **breakout group**, export from inside the group (its Export bar or
the group's options menu) — each group is its own small event, so it gets
its own report.

## What's included — and what isn't

- **Every visible tool, in flow order**, showing what the tool showed:
  results are aggregated and sorted exactly as on screen, charts are drawn
  in, piped data reflects your pipe filters, and Instructions pages appear
  with their formatted content.
- **Hidden tools are left out** — the report is
  what-you-see-is-what-you-get. Unhide a tool before exporting if you want
  it in.
- The Word Cloud section shows the top words, as on screen. If you want the
  full word list — or any tool's data as rows you can sort, filter and
  chart yourself — use the **[CSV export](/facilitator/csv-export.html)**;
  the report is for reading, the CSVs are for working.

## Habits worth having

- **Export at the end of the session, while everything is fresh** — the
  report captures the event as it stands, and it keeps working even after
  the event is deleted.
- Skim it before sending: the report shows tools as participants left them,
  including any unfinished or test entries you may want to tidy first.
