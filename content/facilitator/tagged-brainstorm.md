# Tagged Brainstorming — for facilitators

Tagged Brainstorming is a brainstorm with labelled buckets built in:
participants generate ideas *straight into* the category each idea belongs to,
instead of brainstorming a flat list and sorting it afterwards. Classic uses:
**Helps & Hinders**, pros/cons, "questions for panel A / panel B", risks by
theme.

## How to run it

- In the tool's settings, name your buckets as a comma-separated list (for
  example `Helps, Hinders`). Each bucket becomes a coloured bar; colours are
  assigned automatically from a colour-blind-safe palette.
- Participants open a bucket and add ideas inside it. Everything is visible to
  everyone as it arrives, newest first.
- Anyone can re-file an idea into a different bucket using the selector on the
  item — sorting stays a shared, social act.

## Where the buckets pay off

Each idea carries its bucket as a tag, so downstream tools can fan out by
category: pipe *Helps* into one tool and *Hinders* into another using the
pipe filter's tag picker. One brainstorm, several parallel next steps.

## When to use it (and when not)

- Use it when the categories are known **before** ideas start flowing — the
  buckets steer the group's attention.
- If you want ideas first and themes to *emerge* from them, run a plain
  Brainstorm and follow it with Clustering instead.
- Two to four buckets is the sweet spot; beyond six, choosing a bucket starts
  to compete with having ideas.
