# Five Whys — for facilitators

Five Whys is a root-cause tool: the group drills from a problem to its underlying
causes by repeatedly asking "why," building a collaborative tree. You mark the
true root causes for downstream work.

## How to run it

- State the problem clearly in the **Prompt**.
- Let the group branch; use the outliner view to follow the tree.
- Mark the root causes — they're copied out as items for the next step.

## Facilitation tips

- Keep asking "why" past the first comfortable answer; the real cause is usually
  a few levels down.
- Steer away from blame toward systems and conditions.
